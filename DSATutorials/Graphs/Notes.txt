
1. In Dijikstra : If a -ve edge is there we may loose the O(ELogV) Time and might turn into exponenential. As it will see a -ve edge node and start processing again which is not effecient

2. Bellman : Does not matter if a -ve edge is there or how many of those are there, i will only work for v-1 times.

3. In Prims : the moment you pop a node we dont care further and mark it visited as we are trying to be absolutey connected to its neighbors

4. In Kruskal's : We sort the edges and then consider only v-1 edges in graph with help of union find algorithm to make sure no cycle is found

5. Topo sort : Helps in telling that given a directed graph and 2 edges u and v, where u -> v means u will always come beofre v.
Note : Usually in BFS for topo, if we dont have any node with indegree 0 then we may have a cycle or the count of nodes for graph is not equals to what BFS tells us
